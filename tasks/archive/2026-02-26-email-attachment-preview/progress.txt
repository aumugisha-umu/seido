## Key Learnings
- **Reuse existing RLS helpers**: `get_team_id_from_document_path()` works for any `{team_id}/...` path pattern
- **Double validation pattern**: attachment‚Üíemail + email‚Üíteam gives defense-in-depth with generic 404
- **Signed URL caching for images**: Store in React state to avoid re-fetching on re-render; PDFs open immediately so no cache needed
- **Raw `<img>` for dynamic signed URLs**: `next/image` requires `remotePatterns` + known dimensions; signed URLs from Supabase Storage are dynamic and unpredictable
