# Started: 2026-02-23
# Stories: 1 (bugfix)

## BUG-001: SSR getLinkedEntities showing all entities instead of email-linked only ✅
**Date:** 2026-02-23
**Files:** app/gestionnaire/(with-navbar)/mail/page.tsx (lines 124-290)
**Changes:**
- Replaced 6 direct entity table queries (buildings, lots, contacts, contracts, interventions, companies) with RPC-first approach
- Calls `get_distinct_linked_entities(p_team_id)` to get only IDs present in `email_links` junction table + counts
- Groups by entity_type, fetches details only for linked IDs in parallel via Promise.all
- JS fallback if RPC not deployed: queries email_links directly + deduplicates in JS
- Real emailCount values instead of hardcoded 0
- Added subtitle support for lots (building name) and contacts (email address)
**Root cause:** SSR page.tsx was written as "fetch all entities" shortcut during initial implementation, while API endpoint was later optimized with proper junction-table filtering. The SSR path was never updated.
**Learnings:** SSR initial load and API endpoint must always use the same data-fetch strategy — when adding or optimizing an API endpoint, check if an SSR equivalent exists and update both.
