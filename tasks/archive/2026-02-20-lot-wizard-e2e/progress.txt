## 2026-02-20 — Bugfix: Independent lots address display (3-layer fix)
- Implemented: Fixed address not appearing for independent lots in (1) lots list table and (2) lot detail header
- Root cause: findByTeam() had address JOINs removed during optimization but no batch replacement added; page.tsx required lat/lng coordinates even for text-only address display; client LotAddress type was too narrow
- Files: lot.repository.ts (batch address fetch), page.tsx (relaxed conditions), lot-details-client.tsx (nullable coords + map guard)
- Learnings: When removing nested JOINs for perf, always verify a batch replacement exists. Separate "displayable data" (text) from "feature data" (coordinates) — don't block address display on map-specific fields
- Decision: Used same batch-fetch pattern as findByIdWithRelations() for consistency; made coordinates nullable instead of creating separate types for map vs text display
