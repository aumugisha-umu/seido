## 2026-03-05 14:00 — Dashboard Mobile Stats: CSS @layer Fix
- Implemented: Moved `.dashboard__stats--mobile/--desktop` visibility rules from `@layer components` to unlayered CSS with `!important` in globals.css
- Files: `app/globals.css`, `components/dashboards/manager/manager-dashboard-v2.tsx`
- Learnings: Tailwind v4 `@layer components` has lower priority than `@layer utilities` and unlayered styles. During Next.js hydration, layered responsive rules get overridden. Use unlayered + `!important` for critical layout toggles.
- Decision: BEM modifiers (`.dashboard__stats--mobile/--desktop`) instead of Tailwind responsive classes for deterministic behavior.

## 2026-03-05 15:30 — Contact Creation: Empty Email Duplicate Key Fix
- Implemented: Replaced `UNIQUE NULLS NOT DISTINCT (email, team_id)` with partial unique index `WHERE email IS NOT NULL AND email != ''`. Added Zod preprocess (empty→null) + API normalization.
- Files: `supabase/migrations/20260304100000_fix_users_email_team_unique_allow_null.sql`, `lib/validation/schemas.ts`, `app/api/create-contact/route.ts`
- Learnings: `NULLS NOT DISTINCT` treats NULL as equal. Partial unique index is the correct approach for "unique if present, allow multiple NULLs" semantics.
