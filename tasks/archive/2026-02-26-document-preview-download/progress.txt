## Cross-Cutting Learnings

### API Route vs Client-Side Storage
- `createBrowserSupabaseClient().storage.createSignedUrl()` is unreliable: browser JWT can be stale, RLS checks at CDN level fail silently
- Server-side API routes (`/api/view-intervention-document`, `/api/download-intervention-document`) with `getApiAuthContext()` are reliable and use fresh cookies
- Cross-origin `download` HTML attribute is IGNORED â€” must use `{ download: fileName }` option in `createSignedUrl()` to set `Content-Disposition: attachment` server-side

### Hook-as-Component Pattern
- Returning a pre-wired ReactNode from a custom hook (via `useMemo`) eliminates state/callback prop threading
- Consumer code goes from ~50 lines of wiring to 2 lines: destructure hook + render `{previewModal}`
- Trade-off: less flexible (can't customize modal per-role), but for our use-case identical behavior is the goal

### Duplication Stats
- Before: ~160 lines of duplicate logic across 3 files
- After: 1 shared hook (128 lines) + 2 lines per consumer = ~134 lines total
- Net reduction: ~26 lines, but more importantly: single source of truth
