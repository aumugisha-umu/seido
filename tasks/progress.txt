## 2026-02-18 — Fix: Building Interventions Tab Showing 0 Interventions
- Implemented: New `getByBuildingWithLots(buildingId, lotIds)` method in intervention-service.ts using Supabase `.or()` filter to fetch both building-level AND lot-level interventions in 1 query. Replaced `getByLotIds(lotIds)` call in building detail page.tsx. Removed `if (lots.length > 0)` guard so buildings with 0 lots still show building-level interventions.

- Learnings: XOR constraints (`building_id XOR lot_id`) mean querying only one column misses the other half. SQL `NULL IN (...)` is always UNKNOWN/false. Use `.or()` to combine both sides.
- Decision: Single `.or()` query over two separate queries + merge — simpler, no deduplication needed, consistent response shape.
