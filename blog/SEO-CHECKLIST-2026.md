## 4. Content Structure & Length

### Heading Hierarchy (Critical)

- [ ] **One H1 per article**: Main topic, includes primary keyword
- [ ] **H2 for main sections**: Use for topic groupings
- [ ] **H3 for subsections**: Refinements under H2
- [ ] **Never skip levels**: Go H1 → H2 → H3 → H4 (no H1 → H4 jumps)
- [ ] **Logical flow**: Each heading clarifies relationship to parent topic

**Example Structure:**
```
# H1: Comment Optimiser la Gestion des Interventions Immobilières
## H2: Pourquoi la Gestion Manuelle Pose Problème
### H3: Temps Perdu en Coordination
### H3: Risques d'Erreurs de Communication
## H2: Les Avantages d'une Plateforme Centralisée
### H3: Suivi en Temps Réel
### H3: Automatisation des Notifications
```

### Content Length

- [ ] **Target**: 1,500+ words for ranking content (average #1 result on Google)
- [ ] **Principle**: Fully satisfy user's search intent - no question left unanswered
- [ ] **Section length**: 300-500 words per section
- [ ] **Balance**: Break dense sections into H3 subsections for readability
- [ ] **Quality over quantity**: Comprehensive value more important than arbitrary word count

### Formatting for Readability

- [ ] Short paragraphs (3-5 lines maximum)
- [ ] Bullet points for lists
- [ ] Bold key phrases for scanning
- [ ] Include visual breaks (images, tables, code blocks)
- [ ] Use numbered lists for step-by-step processes

### Internal Linking

- [ ] Link to related blog posts (3-5 internal links minimum)
- [ ] Link to product pages where relevant
- [ ] Use descriptive anchor text (not "click here")
- [ ] Include contextual links within body content

### CTA Placement

- [ ] **Top**: Brief CTA after intro (low-pressure)
- [ ] **Middle**: Contextual CTA within content
- [ ] **End**: Primary CTA after conclusion (strong ask)
- [ ] Make CTAs specific to article topic

## 6. Featured Snippet Optimization

### Understanding the 2026 Landscape

- [ ] **Priority**: Focus on pages already ranking on page 1 (99% of snippets come from top 10)
- [ ] **AI Integration**: Optimize for Search Generative Experience (SGE) and AI Overviews
- [ ] **Entity Authority**: Establish brand as leading voice in PropTech property management

### Answer-First Approach

- [ ] **Put answer in first paragraph**: Direct answer within first 40-60 words
- [ ] **Question as H2**: Use target question as section heading
- [ ] **Immediate value**: Don't make readers (or AI) dig for the answer

**Example:**
```markdown
## Comment Automatiser la Gestion des Interventions Immobilières ?

L'automatisation de la gestion des interventions immobilières se fait
en centralisant les demandes, les devis et le suivi dans une plateforme
unique qui notifie automatiquement les prestataires, suit les délais,
et archive les documents pour un historique complet.
```

### Content Formats for Snippets

**Paragraph Snippets (Definitions):**
- [ ] 40-60 words
- [ ] Direct answer to "What is..." or "How to..." questions
- [ ] Use definition format: "[Term] is [definition]"

**List Snippets (Steps/Rankings):**
- [ ] Use numbered lists for processes/steps
- [ ] Use bullet points for features/benefits
- [ ] 3-8 items optimal
- [ ] Keep each item concise (1-2 lines)

**Table Snippets (Comparisons):**
- [ ] Use markdown tables for comparisons
- [ ] Include price comparisons, feature comparisons, pros/cons
- [ ] Keep tables clean and scannable
- [ ] 3-5 columns maximum

**Example Table:**
```markdown
| Critère | Gestion Manuelle | Seido |
|---------|------------------|-------|
| Temps de traitement | 2 heures | 15 minutes |
| Coût mensuel | 500€ (temps perdu) | 99€ |
| Taux d'erreur | 15% | 2% |
```

### Semantic Structure

- [ ] Use clear H2/H3 headings with question format
- [ ] Structure content with semantic HTML (lists, tables, definitions)
- [ ] Include schema markup (FAQ, HowTo, Article)
- [ ] Use descriptive subheadings that guide AI extraction

### Authority Signals for Snippets

- [ ] **E-E-A-T matters**: Anonymous content rarely wins snippets in 2026
- [ ] Include author credentials
- [ ] Cite authoritative sources
- [ ] Link to related expert content

### Monitoring & Iteration

- [ ] Track snippet ownership with Ahrefs/SEMrush
- [ ] Monitor keywords where you currently own snippets
- [ ] Identify newly acquired snippet keywords
- [ ] Track recently lost snippets and optimize

## 7. Markdown Frontmatter Metadata

### Essential SEO Fields

```yaml
---
# Page Metadata
title: "Primary Title with Target Keyword (50-60 chars)"
description: "Compelling meta description with primary + secondary keywords (150-160 chars)"
slug: "clean-url-slug-with-keywords"
date: "2026-02-19"
updated: "2026-02-19"

# SEO Configuration
seo:
  title: "Optimized Title for SERP | Seido"
  description: "Custom meta description if different from main description"
  keywords: ["keyword1", "keyword2", "keyword3"]
  canonical: "https://seido.be/blog/slug"
  robots: "index, follow"

# Content Classification
category: "Gestion Immobilière"
tags: ["interventions", "automatisation", "PropTech"]
author: "Author Name"
lang: "fr-BE"

# Schema/Structured Data
schema:
  type: "BlogPosting"
  image: "/images/featured-image.jpg"

# Social Media
og:
  title: "Social Media Title (Different from SEO title)"
  description: "Social media description"
  image: "/images/og-image.jpg"
  type: "article"

twitter:
  card: "summary_large_image"
  title: "Twitter Title"
  description: "Twitter description"
  image: "/images/twitter-image.jpg"
---
```

### Field-by-Field Best Practices

**Title:**
- [ ] 50-60 characters (display limit in SERPs)
- [ ] Include primary keyword near beginning
- [ ] Lead with buyer's problem/benefit
- [ ] Front-load important words (Google may rewrite 60-70%)

**Description:**
- [ ] 150-160 characters optimal
- [ ] Include primary + secondary keywords naturally
- [ ] Treat as "mini sales pitch"
- [ ] Include call-to-action or value proposition
- [ ] Note: Google rewrites 60-70% of meta descriptions, but still important

**Slug:**
- [ ] kebab-case format
- [ ] Include primary keyword
- [ ] Keep short (3-5 words)
- [ ] Avoid stop words (le, la, des, etc.) unless needed for clarity
- [ ] Use hyphens, not underscores

**Keywords Array:**
- [ ] 3-8 relevant keywords
- [ ] Mix of primary, secondary, long-tail
- [ ] YAML array format: `["keyword1", "keyword2"]`
- [ ] Helps content management, less important for modern SEO

**Tags:**
- [ ] 3-6 tags per article
- [ ] Use for internal categorization and content clusters
- [ ] Keep consistent taxonomy across blog
- [ ] Link to tag archive pages

**Author:**
- [ ] Full author name (matches author bio)
- [ ] Link to author page/bio
- [ ] Use same author for related articles (builds topical authority)

**Date & Updated:**
- [ ] ISO 8601 format: `YYYY-MM-DD`
- [ ] Display "Last updated" date prominently
- [ ] Update when making significant content revisions
- [ ] Signals freshness to Google

**Language:**
- [ ] Use `fr-BE` for Belgian French
- [ ] Helps search engines understand target region
- [ ] Important for multilingual sites

**Canonical URL:**
- [ ] Full absolute URL to prevent duplicate content issues
- [ ] Use for cross-posting or similar content

**Robots:**
- [ ] Default: `"index, follow"`
- [ ] Use `"noindex, nofollow"` for thin/duplicate content
- [ ] Use `"index, nofollow"` for user-generated content

### Data Type Best Practices

- [ ] **Be explicit**: Strings in quotes, numbers without quotes
- [ ] **Arrays**: Use YAML array syntax `["item1", "item2"]`
- [ ] **Booleans**: `true` or `false` (lowercase, no quotes)
- [ ] **Dates**: ISO format `"2026-02-19"`
- [ ] **Nested objects**: Use proper YAML indentation

### Validation

- [ ] Use YAML validator before publishing
- [ ] Test frontmatter parsing in development
- [ ] Verify all required fields are present
- [ ] Check for syntax errors (unclosed quotes, incorrect indentation)

## 8. Technical SEO Checklist

### Page Speed & Core Web Vitals

- [ ] **LCP (Largest Contentful Paint)**: < 2.5s
- [ ] **FID (First Input Delay)**: < 100ms
- [ ] **CLS (Cumulative Layout Shift)**: < 0.1
- [ ] Optimize images (WebP format, lazy loading)
- [ ] Minimize JavaScript/CSS
- [ ] Use CDN for assets

### Mobile Optimization

- [ ] Responsive design for all screen sizes
- [ ] Touch-friendly buttons (minimum 48x48px)
- [ ] Readable font sizes (minimum 16px)
- [ ] No horizontal scrolling
- [ ] Test on real mobile devices

### Structured Data (Schema.org)

- [ ] **Article schema**: BlogPosting or Article
- [ ] **Author schema**: Person with name, image, URL
- [ ] **Organization schema**: Logo, contact info
- [ ] **FAQ schema**: For Q&A sections
- [ ] **HowTo schema**: For step-by-step guides
- [ ] **Breadcrumb schema**: For navigation
- [ ] Test with Google Rich Results Test

### Image Optimization

- [ ] Descriptive file names (gestion-interventions-seido.jpg)
- [ ] Alt text with keywords (but natural)
- [ ] Compressed file sizes (< 200KB)
- [ ] Appropriate dimensions (no oversized images)
- [ ] WebP format with JPG fallback
- [ ] Lazy loading for below-fold images

### URL Structure

- [ ] Clean, descriptive URLs
- [ ] Include target keyword
- [ ] Use hyphens for word separation
- [ ] Keep URLs short (< 100 characters)
- [ ] Avoid parameters/session IDs
- [ ] Implement proper redirects for changed URLs

### Security & Trust

- [ ] HTTPS (SSL certificate)
- [ ] Valid SSL certificate (no warnings)
- [ ] Privacy policy page
- [ ] Terms of service page
- [ ] Cookie consent banner (GDPR compliance)
- [ ] Contact information easily accessible

## 9. Content Publishing Workflow

### Pre-Publishing Checklist

**Content Quality:**
- [ ] Article meets minimum word count (1,500+ for ranking content)
- [ ] All sections have proper heading hierarchy
- [ ] No grammatical/spelling errors (run through Grammarly/LanguageTool)
- [ ] Tone matches brand voice (professional but approachable)
- [ ] Includes original insights/examples from Seido

**SEO Optimization:**
- [ ] Primary keyword in H1, first paragraph, conclusion
- [ ] Secondary keywords distributed naturally throughout
- [ ] Meta title and description optimized
- [ ] Frontmatter complete and validated
- [ ] Internal links to 3-5 related articles
- [ ] External links to authoritative sources (2-3)
- [ ] Images optimized and have alt text

**E-E-A-T:**
- [ ] Author bio included with credentials
- [ ] Publication/updated dates present
- [ ] Data/claims backed by sources
- [ ] Screenshots/examples demonstrate first-hand experience

**Featured Snippet:**
- [ ] Answer-first paragraph for target question
- [ ] Structured content (lists, tables, definitions)
- [ ] Clear H2 headings with question format

**Technical:**
- [ ] Page speed tested (< 3s load time)
- [ ] Mobile-friendly (test on device)
- [ ] Schema markup implemented
- [ ] Images compressed and lazy-loaded
- [ ] No broken links (internal or external)

### Post-Publishing Tasks

**Week 1:**
- [ ] Submit URL to Google Search Console for indexing
- [ ] Share on social media (LinkedIn, Twitter, Facebook)
- [ ] Send to email newsletter subscribers
- [ ] Monitor for initial indexing (check Google)

**Week 2-4:**
- [ ] Monitor Google Search Console for impressions/clicks
- [ ] Track ranking position for target keywords
- [ ] Respond to any comments/questions
- [ ] Share in relevant LinkedIn/Facebook groups

**Month 2-3:**
- [ ] Analyze performance data (traffic, rankings, conversions)
- [ ] Identify opportunities to improve
- [ ] Update content if needed (refresh data, add new sections)
- [ ] Build backlinks through outreach

**Ongoing:**
- [ ] Quarterly content audit (update stats, examples, screenshots)
- [ ] Monitor for keyword ranking changes
- [ ] Refresh meta descriptions if CTR is low
- [ ] Add new internal links as related content is published

### Content Promotion Strategy

- [ ] Share on Seido social media channels
- [ ] Include in email newsletter
- [ ] Post in relevant Belgian real estate forums/groups
- [ ] Reach out to influencers for shares
- [ ] Consider paid promotion (LinkedIn ads) for high-value content
- [ ] Repurpose into other formats (infographic, video, podcast)

### Performance Metrics to Track

- [ ] **Organic traffic**: Google Analytics
- [ ] **Keyword rankings**: Ahrefs/SEMrush
- [ ] **Click-through rate**: Google Search Console
- [ ] **Engagement**: Time on page, bounce rate, scroll depth
- [ ] **Conversions**: Demo requests, sign-ups from blog
- [ ] **Featured snippets**: Monitor owned snippets
- [ ] **Backlinks**: Track referring domains

## Quick Reference Summary

### Top 5 Priorities for Every Article

1. **E-E-A-T**: Author bio, original examples, first-hand experience, citations
2. **Heading Hierarchy**: One H1, logical H2/H3 structure, no level skipping
3. **Featured Snippets**: Answer-first paragraph, structured content (lists/tables), question-format headings
4. **French-Belgian Optimization**: Local keywords, Belgian terminology, regional examples
5. **Frontmatter Metadata**: Complete SEO fields, proper YAML formatting, optimized title/description

### Common Mistakes to Avoid

- Skipping heading levels (H1 → H4)
- Anonymous content without author credentials
- Burying the answer deep in content (put it first)
- Generic keywords without Belgian context
- Incomplete or invalid frontmatter
- Content shorter than 1,500 words for competitive keywords
- Missing internal links to related articles
- No clear CTA or conversion path

### Tools to Use

- **Keyword Research**: Ahrefs, SEMrush (filter by Belgium/French)
- **Content Optimization**: Surfer SEO, Clearscope
- **Grammar/Spelling**: Grammarly, LanguageTool (for French)
- **Page Speed**: Google PageSpeed Insights, GTmetrix
- **SEO Audit**: Google Search Console, Screaming Frog
- **Schema Validation**: Google Rich Results Test
- **YAML Validation**: yamllint.com
