## Learnings
- vi.hoisted() is REQUIRED for mock variables used inside vi.mock() factories (hoisting issue)
- Class expression in vi.mock() for constructor mocks (vi.fn().mockImplementation fails)
- Windows filename length limits break git worktree with deep nesting — run agents without worktree for deep paths
- Promise.allSettled for bulk operations (bank account creation, rent call generation) — resilient to partial failures
- Exported pure functions from services (calculateMatchConfidence, calculateDueDates, isTokenExpiringSoon) enable testing without mocking
