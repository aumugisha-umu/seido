## 2026-02-08 — US-001 ✅

**Story:** Fix N+1 queries dans use-tenant-data

**Changes:**
- Modified `hooks/use-tenant-data.ts` (lines 220-270)
- Replaced Promise.all(map) with 3 batch queries using `.in('intervention_id', ids)`
- Added Map<string, T[]> for O(1) lookup instead of repeated filtering

**Performance Gain:**
- Before: 3N queries (3 queries per intervention)
- After: 3 queries total (regardless of intervention count)
- Example: 10 interventions = 30 → 3 queries (90% reduction)

**Learnings:**
- Pattern: Extract IDs first, batch query with `.in()`, then Map for fast lookup
- Always include `intervention_id` in SELECT for mapping back to parent
- Handle empty array case to avoid `.in()` with empty array
